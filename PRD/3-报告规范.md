# 📊 报告输出界面（Report UI/UX）变动需求

## 1. 核心指标的呈现方式变动

原先的报告可能使用了“分数”或“百分比”来展示表现。现在，必须引入“**标准分 (T-Score)**”和“**常模百分位 (Percentile Rank)**”来展示专业性。

| 原有指标描述 | 新增/修改指标描述 | 目的 |
| :--- | :--- | :--- |
| 反应速度分数 (如 95 分) | **平均反应时 T 分数** ($T_{RT}$) | 明确指标是经过常模校正的标准分。 |
| 分数变异性 (如 90 分) | **反应时变异性 T 分数** ($T_{SD}$)| 保持术语专业性。 |
| 准确度 $FAQ$ / 冲动 $FRCQ$ | **漏报 T 分数** ($T_{OM}$) / **误报 T 分数** ($T_{CM}$) | 直接对应测试中的核心行为，避免使用自定义缩写。 |
| 最终 $S$ 概率值 | 保持 $S$ 概率值，但必须**在附近说明其计算基础**。 | 确保用户理解 $S$ 值不再是固定公式，而是基于常模 T 分数。 |

### 1.1 报告指标列表变动示例

| 指标名称 | 用户得分 (T-Score) | 常模对比 (百分位) | 评级/解读 |
| :--- | :--- | :--- | :--- |
| **注意力持续力 ($T_{OM}$)** | 48 | 42th | 正常范围 |
| **反应速度 ($T_{RT}$)** | 35 | 5th | **显著落后 (⚠️)** |
| **冲动控制 ($T_{CM}$)** | 55 | 70th | 高于平均水平 |
| **稳定性 ($T_{SD}$)** | 42 | 19th | 边缘范围 |

## 2. 引入可视化常模对比图（关键！）

为了让用户直观理解 $T$ 分数的含义，我们必须引入**常模分布图**。

*   **图表类型：** **标准正态分布曲线图**或简化的柱状图。
*   **展示内容：**
    1.  显示标准分的分布（均值 $T=50$，标准差 $T=10$）。
    2.  突出显示**用户当前 $T$ 分数**所在的位置。
    3.  标示出**正常范围** (40-60分)、**边缘范围** (30-40分/60-70分) 和**风险范围** (低于 30 分)。

> **产品经理要求：** 在报告中，用户看到 $T_{RT} = 35$ 时，必须能直观看到它落在了常模曲线的左侧（即 **5% 风险区**），而不是一个空泛的数字。

## 3. 报告解读部分的变动

解读内容必须结合用户输入的基础信息。

### 3.1 报告标题/基本信息

在报告顶部，必须显示用于计算常模校正的原始数据。

> 📄 **报告基本信息**
>
> **测试对象:** [用户输入的性别]
> **常模校正年龄组:** [根据用户年龄划分的年龄组，例如：18-40岁青年组]
> **测试时长:** [XX 分 XX 秒]
> **测试时间:** [日期/时间]

### 3.2 最终 $S$ 概率解读

针对最终的 $S$ 概率值，必须增加一句解释，强调其基于常模。

> **核心风险指数 ($S$): [XX%]**
>
> **解读说明：** 该指数反映了测试对象在各项指标上，**相较于同年龄、同性别常模群体**的偏离程度。如果该值较高，表示在注意力或冲动控制方面表现出与常模**显著不同**的行为模式。
